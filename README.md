# blind-signature
Module for server side & application for user client for interaction with blind signature

### Papers for Reference
- [RFC 9474](https://www.rfc-editor.org/rfc/rfc9474.pdf)
- [Gentle Introduction to Blind signatures](https://arxiv.org/html/2509.02189v1)

- [Метод слепой электронной подписи на эллиптических кривых: условия и требования ](https://cyberleninka.ru/article/n/metod-slepoy-elektronnoy-podpisi-na-ellipticheskih-krivyh-usloviya-i-trebovaniya) \[RUS\]
- [ПРОТОКОЛЫ СЛЕПОЙ ЦИФРОВОЙ ПОДПИСИ НА ОСНОВЕ СКРЫТОЙ ЗАДАЧИ ДИСКРЕТНОГО ЛОГАРИФМИРОВАНИЯ](https://cyberleninka.ru/article/n/protokoly-slepoy-tsifrovoy-podpisi-na-osnove-skrytoy-zadachi-diskretnogo-logarifmirovaniya) \[RUS\]



### Terms

| Term | RFC 9474 notation                   | Meaning                                                                                                                |
|------|---------------------------------|------------------------------------------------------------------------------------------------------------------------|
| `pk` | **p**ublic **k**ey              | Generated by server.                                                                                                   |
| `sk` | **s**ecret **k**ey              | Also generated by server, but it shouldn't be published.                                                               |
| `msg` | **m**e**s**sa**g**e             | Some message from user. It can be the choice for some candidate, or just "Yes" etc.                                    |
| `blinded_msg` | **blinded** **m**e**s**sa**g**e | Encrypted message from user. It cannot be decrypted by anybody else, but can be signed by server.                      |
| `inv` | **inv**erse                     | Inverse of blinding factor.                                                                                            |
| `blind_sig` | **blind** **sig**nature         | Signed message. Yay, now you may make your vote!                                                                       |
| `sig` | **sig**nature                   | Encrypted message, which user sends to server with his vote. This post should be anonymous, but server узнает своих :) |
